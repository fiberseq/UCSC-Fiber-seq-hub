Description

Fiber-seq Inferred Regulatory Elements (FIRE)

This track provides Fiber-seq FIRE peak calls and chromatin accessibility for every sample and track type in this hub, across a compendium of cell lines and tissues. If you are unfamiliar with Fiber-seq please see the references below for a detailed description, but in short it is useful to think of it as a long-read, single-molecule, haplotype-resolved version of DNaseI/ATAC-seq that can be used to identify regions of chromatin accessibility. For a quick multi-sample overview, see the default Fiber-seq Accessibility track.

Questions or problems with this hub? Contact the Vollger Lab (mrvollger@genetics.utah.edu) or the Stergachis Lab (absterga@uw.edu).

Display Conventions

Click "Fiber-seq Compendium" to open its configuration page, use the Cell Type drop-down to narrow the sample list to a tissue/cell line of interest, then check the boxes in the sample (columns) x track type (rows) grid for the combinations you want to display, exactly as in ENCODE's composite tracks. Where a track is split by haplotype, haplotype 1 is blue, haplotype 2 is red, and both haplotypes combined ("all") is black, consistently across every sample.

Track Descriptions

  • Percent accessible: The percent of fibers that contain a FIRE element overlapping a given position in the genome, computed separately for haplotype 1, haplotype 2, and both haplotypes combined.
  • Peaks: FIRE peaks are called by identifying FIRE score local-maxima that have FDR values below a 5% threshold. Once a local-maxima is identified, the start and end positions of the peak are determined by the median start and end positions of the underlying FIRE elements.

    • Methods

    FIREs are Modification Sensitive Patches (MSPs) on Fiber-seq reads that are inferred to be regulatory elements on single chromatin fibers, identified using semi-supervised machine learning (Vollger, Swanson et al.). Every individual FIRE element is associated with a precision value, which indicates the probability that the FIRE element is a regulatory element. Please refer to https://fiberseq.github.io/ for more details.
